1. How to Update Your Researcher Profile

Your researcher profile lives in backend/config.json and controls how Claude evaluates every grant. When your situation changes, the profile needs to reflect that.

Using Claude Code (recommended)

Open your terminal, navigate to the grant-radar folder, and run claude. Then describe what you want to change in plain English:

"I've submitted my PhD thesis. Update my profile to show I'm awaiting defence, with submission date October 2026."
"I've successfully defended my PhD as of February 2027. Update my profile to show completed PhD."
"I've accepted a postdoc at Cardiff starting January 2027. Update my priorities — Cardiff is no longer a target, shift focus to fellowship opportunities and Dutch positions."
"Add Japanese as a working language at JLPT N2 level."
"Add a new publication: [title], published in [journal], DOI: [doi]. Methods used: [methods]."

Claude Code will edit config.json for you, adjusting the profile, eligibility rules, timeline, and tier definitions as needed.

Editing directly

For simple changes (dates, keywords, language levels), you can edit backend/config.json directly. Key sections:

  • researcher.academic_status — position, dates, funding
  • researcher.languages — language proficiency levels
  • researcher.skills — array of research skills
  • researcher.eligibility — nationality, degree flags
  • timeline — PhD submission, defence, ideal start window
After editing config.json, commit and push the changes. The next pipeline run will use the updated profile.

2. How to Add or Remove RSS Feeds

RSS feeds are checked daily by the pipeline. They live in backend/config.json under sources.rss_feeds.

Using Claude Code

"Add this RSS feed as a new source: Nature Careers Economics, https://www.nature.com/natcareers.rss"

Editing directly

Add an entry to the rss_feeds array in config.json:

{
  "name": "Nature Careers Economics",
  "url": "https://www.nature.com/natcareers.rss",
  "enabled": true
}

To disable a feed without removing it, set "enabled": false.

Finding RSS feed URLs

  • Many job boards have an RSS icon or link in their footer
  • Try appending /rss or /feed to a job board URL
  • Ask Claude Code: "Find the RSS feed URL for [website name]"

3. How to Add a Web Scraper

Web scrapers are used for sites that don't offer RSS feeds. Adding a new one requires writing Python code, so Claude Code is the way to go.

Using Claude Code

"Add this website as a scraped source: https://www.oecd.org/careers/. Figure out the page structure and write a scraper for it."

Claude Code will:

  1. Examine the website structure
  2. Write a scraper function in backend/fetchers/web_scraper.py
  3. Add the source to config.json
  4. Test the scraper to make sure it works

When a scraper breaks

Websites occasionally redesign, which breaks scrapers. If a source shows a red dot on the Sources tab, tell Claude Code:

"The EURAXESS scraper is broken. It's showing an error on the dashboard. Fix it."

Claude Code will inspect the error, check what changed on the website, and update the scraper code accordingly.

4. How to Modify Priority Tiers

Tiers control how grants are categorised, which ones trigger immediate email alerts, and how they appear on the dashboard. The current tiers are:

  • Tier 1 — Cardiff + Welsh Economic Geography (immediate alert)
  • Tier 2 — Strong Alternatives: Cardiff other depts, Netherlands, PoliMi
  • Tier 3 — Good European Opportunities
  • Tier 4 — Indonesia: BRIN & UGM

Using Claude Code (for structural changes)

"Add a new Tier 2b for Japanese universities, specifically Nagoya and Kyoto economics departments."
"Merge Tier 3 and Tier 4 into a single 'International Opportunities' tier."
"I've accepted a position at Cardiff. Remove Tier 1 and make Dutch universities my new Tier 1."

Editing directly

Edit the tiers section in config.json. Each tier has:

{
  "1": {
    "name": "Cardiff + Welsh Economic Geography",
    "notification": "immediate_email",
    "description": "Cardiff School of Geography...",
    "keywords": ["Cardiff", "Wales", "Welsh"],
    "portable_fellowships": true
  }
}

The notification field controls alerting: immediate_email for Tier 1, weekly_digest_prominent for Tier 2, weekly_digest for the rest.

{
  "models": {
    "daily_analysis": "claude-sonnet-4-6",
    "weekly_discovery": "claude-opus-4-6",
    "weekly_reeval": "claude-opus-4-6"

6. How to Set Up Email Notifications

Grant Radar sends two types of emails:

  • Tier 1 alerts — sent immediately when a top-tier match is found
  • Weekly digest — sent every Sunday with all tiers + strategic notes

Step 1: Create a Gmail App Password

You need a Gmail account with 2-Step Verification enabled.

  1. Go to Google Account > Security
  2. Under "Signing in to Google", click 2-Step Verification (enable it if not already)
  3. At the bottom of the 2-Step Verification page, click App passwords
  4. Enter a name (e.g., "Grant Radar") and click Generate
  5. Copy the 16-character password that appears
Use the generated App Password, not your regular Google password. Regular passwords will not work with SMTP.

Step 2: Add secrets to your GitHub repository

In your GitHub repository:

  1. Go to Settings > Secrets and variables > Actions
  2. Click New repository secret for each:
Secret NameValue
ANTHROPIC_API_KEY Your Claude API key from console.anthropic.com
EMAIL_ADDRESS Your Gmail address (e.g., you@gmail.com)
EMAIL_PASSWORD The 16-character App Password from Step 1

Step 3: Set workflow permissions

In your GitHub repository:

  1. Go to Settings > Actions > General
  2. Under "Workflow permissions", select Read and write permissions
  3. Click Save

This allows the pipeline to commit updated data files back to the repo.

7. How to Run Manually

From GitHub (easiest)

  1. Go to your repository on GitHub
  2. Click the Actions tab
  3. Select Daily Grant Collection or Weekly Deep Search
  4. Click Run workflow > Run workflow
  5. Wait for the run to complete (daily: ~10 min, weekly: ~30 min)

From your terminal

Make sure you have the dependencies installed:

cd grant-radar
pip install -r requirements.txt

Run the daily pipeline:

ANTHROPIC_API_KEY=sk-ant-... python backend/main.py --mode daily

Run the weekly pipeline:

ANTHROPIC_API_KEY=sk-ant-... python backend/main.py --mode weekly

Dry run (tests fetchers without calling Claude or sending emails):

python backend/main.py --mode daily --dry-run

On Windows, set the environment variable with set ANTHROPIC_API_KEY=sk-ant-... before the Python command, or use PowerShell: $env:ANTHROPIC_API_KEY="sk-ant-..."

8. Troubleshooting

A source shows red on the Sources tab

The scraper for that source is likely broken due to a website redesign. Open your terminal in the grant-radar folder, run claude, and say: "The [source name] scraper is broken, fix it." Claude Code will inspect the error, check the website, and update the scraper.

No new grants found for several days

Check if RSS feed URLs are still valid by visiting them in your browser. Check the Sources tab for error messages. If sources look healthy but no grants appear, it may simply be a quiet period — some feeds only update weekly.

Emails not sending

Check these in order:

  1. Verify EMAIL_ADDRESS and EMAIL_PASSWORD are set correctly in GitHub repository secrets
  2. Make sure you're using a Gmail App Password, not your regular Google password
  3. Check your spam folder — the first email may be flagged
  4. Check GitHub Actions logs for SMTP error messages
API errors in the pipeline

Usually caused by an invalid or expired API key, or insufficient credit balance.

  1. Check your API key at console.anthropic.com/settings/keys
  2. Check your credit balance at console.anthropic.com/settings/billing
  3. Make sure the ANTHROPIC_API_KEY secret in your GitHub repo matches your current key
Dashboard shows old data

The dashboard reads from data/grants.json, which is updated when the pipeline runs and commits. Check:

  1. Go to GitHub > Actions tab > check if recent runs succeeded or failed
  2. If runs failed, click on the failed run to see the error logs
  3. Check that workflow permissions are set to "Read and write" (Settings > Actions > General)
Pipeline crashes or hangs

Check data/last_run_summary.json for the most recent pipeline summary, including error counts and messages. If the pipeline keeps crashing, run claude and say: "The daily pipeline is crashing. Check the Actions logs and fix it."

Grants are being scored incorrectly

The scoring is driven by your researcher profile in config.json. If grants are consistently mis-scored, your profile may need updating. Run claude and describe the problem: "Grants about urban planning are scoring too high — I want to focus more on regional economics."
